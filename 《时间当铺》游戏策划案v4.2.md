# 《时间当铺》游戏策划案 v4.2

> 叙事解谜 / 像素风 / HTML5 Canvas
> 版本：v4.2（架构优化版 — 数据驱动重写 + 跨周目系统文档化 + 叙事设计深化）
> 制作：盛洁

---

## 一、项目概述

### 1.1 一句话定位
经营一家收购"时间"的当铺，通过改写客人的过去来改变现在。

**差异化标签**：跨时空蝴蝶效应 × 数据驱动叙事 × 像素风叙事解谜

### 1.2 类型与风格
| 维度 | 内容 |
|------|------|
| 类型 | 叙事解谜 + 经营模拟 |
| 画面 | 像素风（16×20精灵，程序化背景） |
| 平台 | HTML5 Canvas，纯浏览器运行，无需安装 |
| 技术栈 | 原生 JavaScript，无框架依赖 |
| 存档 | localStorage（局内存档 + 跨周目 MetaSave 持久化）|

### 1.3 核心体验
玩家扮演一家神秘当铺的老板。客人以"记忆片段"作为抵押，换取改变命运的机会。进入客人的记忆，在关键节点做出选择——每个选择产生"时间涟漪"，跨越时空影响其他客人的命运。

### 1.4 核心设计亮点（v4.2 新增）

**亮点一：数据驱动的叙事引擎**
所有叙事内容（对话、选择、结局、蝴蝶效应、隐藏事件）均定义在 `data.js` 配置文件中，逻辑层（`game.js`）仅负责读取数据、执行效果。增加新角色/对话无需修改逻辑代码。

**亮点二：跨周目 MetaSave 系统**
游戏引入双层存档架构：局内存档（当铺状态）+ MetaSave（跨周目持久化）。MetaSave 记录所有结局、涟漪、成就、收藏品，支持"新周目+"特性——玩家在不同周目间共享认知，体验随周目增加而解锁的内容递进。

**亮点三：多层次因果网络**
7 位主线角色 + 1 位隐藏角色的命运通过 15 条涟漪、6 组共振效应、12+ 隐藏事件编织成交叉因果网，每个选择都有跨越时空的回响。

## 二、核心机制

### 2.1 时间交易循环

```
每日开始
  ↓
接待客人 → 进入记忆 → 二选一抉择 → 产生涟漪 → 获得时间碎片
  ↓
经营当铺（升级设施 / 延长时间）
  ↓
结束今天 → 时间碎片 -5 → 触发蝴蝶效应检测
  ↓
30天时限内完成所有客人故事 → 触发结局
```

### 2.2 数据驱动的叙事系统

**设计原则：内容与逻辑解耦**

```javascript
// 逻辑层（game.js）— 通用处理，不含具体内容
class MemoryScene {
  _buildChoices() {
    const choices = [...this.storyData.choices];  // 读取数据
    if (this.storyData.hiddenChoice) {
      const allMet = this.storyData.hiddenChoice.condition.every(
        r => this.game.state.hasRipple(r)
      );
      if (allMet) choices.push(this.storyData.hiddenChoice.choice);
    }
    return choices;
  }
}

// 数据层（data.js）— 定义所有叙事内容
const STORY_DATA = {
  clientId: {
    choices: [
      { text: "从军", rippleId: "chen_army", result: [...] },
      { text: "学医", rippleId: "chen_doctor", result: [...] }
    ],
    hiddenChoice: {
      condition: ["chen_army", "chen_doctor"],  // 跨周目双选解锁
      choice: { text: "???", rippleId: "hidden_path" }
    }
  }
}
```

**数据驱动覆盖范围：**
- 问候语变体（蝴蝶效应影响）
- 记忆场景对话链
- 选择结果与涟漪
- 结局对话
- 离场对话（conditionalDeparture）
- 隐藏事件触发条件

### 2.3 蝴蝶效应系统

**四种蝴蝶效应类型：**

| 类型 | 机制 | 示例 |
|------|------|------|
| `block_and_replace` | 涟漪触发 → 角色被阻断 → 替换为蝴蝶效应对话 | 陈伯学医 → 林小姐母亲被救 → 林小姐不来当铺 |
| `greeting_variant` | 涟漪触发 → 替换角色问候语 | normal/butterfly 两种问候 |
| `ripple_cross_link` | 涟漪组合 → 解锁隐藏选项/成就 | 全部涟漪解锁隐藏角色 |
| `meta_cross_link` | 跨周目涟漪 → 解锁跨周目内容 | 新周目时先生三形态 |

**共振涟漪（6组）：**
特定涟漪组合同时存在于当前周目时，触发共振效果，获得额外时间奖励 + 收藏品。

```
共振条件：涟漪A && 涟漪B && 涟漪C（可跨周目）
    ↓
共振触发：时间碎片奖励 + 收藏品（部分）
    ↓
MetaSave 记录：allResonancesSeen[resId] = true
```

### 2.4 跨周目 MetaSave 系统

**双层存档架构：**

```
┌─────────────────────────────────────────────────┐
│ 局内存档 (localStorage: timePawnshop)           │
│ ├── 时间碎片 / 当铺等级 / 已完成客人              │
│ ├── 当前周目涟漪 / 已购设施                      │
│ └── 本周目统计（停留帧数、犹豫时间等）            │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│ MetaSave (localStorage: timePawnshop_meta)       │
│ ├── achievements: {}  // 已解锁成就 + 解锁时间     │
│ ├── endingsSeen: {}  // 已达成结局               │
│ ├── allRipplesSeen: {}  // 跨周目全部涟漪       │
│ ├── allChoicesMade: {}  // 跨周目全部选择        │
│ ├── allResonancesSeen: {}  // 跨周目全部共振    │
│ ├── collectibles: {}  // 已收集信物              │
│ ├── hiddenEventsTriggered: {}  // 隐藏事件      │
│ ├── titles/sprites/themes: []  // 已解锁奖励     │
│ └── trueEndingSeen / shixianshengChoices 等      │
└─────────────────────────────────────────────────┘
```

**新周目+逻辑：**
```
totalPlaythroughs > 0 → 播放新周目彩蛋对话
isNewGamePlus = true
```

**存档兼容：**
MetaSave._load() 中自动对比 _default() 填充缺失字段，无需版本迁移逻辑。

### 2.5 分支问候系统（branchingGreeting）

**两种分支模式：**

```
直接分支（阿蘅/老吴/小周）：
  问候语播放 → 立即进入分支选择
  ┌──────────────────────────────┐
  │ "祖父是守钟人？"              │
  │ "'等待'怎么典当？"            │
  │ "你手腕上的红绳……"           │
  └──────────────────────────────┘

延迟分支（时先生）：
  基础问候（baseGreeting）→ 完整介绍 → 再进入分支选择
  ┌──────────────────────────────────────┐
  │ [完整介绍：时先生的身份与沙漏的来历]  │
  │                                      │
  │ 接过沙漏？  还是打碎它？              │
  └──────────────────────────────────────┘
```

## 三、叙事设计

### 3.1 四层叙事结构

```
表层叙事：帮助客人解决人生遗憾（单元剧）
    ↓
里层叙事：老板自身的记忆被封锁（主线）
    ↓
深层叙事：所有客人命运交织同一条时间线（闭环）
    ↓
元叙事：玩家在不同周目的选择本身构成故事（MetaSave 驱动）
```

### 3.2 角色命运网络

**时间跨度：** 1962年（陈伯岔路口）→ 1985年（林小姐母亲）→ 1989年（老吴暗房）→ 1997年（阿蘅钟楼）→ 2010年（小周邮筒）→ ????年（时先生当铺创立）→ 当下（玩家掌柜）

**核心联系：**

```
陈伯选择学医（1985）→ 成为外科医生 → 救了林小姐母亲
                                         ↓ 蝴蝶效应阻断
林远（林小姐之父）→ 去世前寄出遗信（2010前）→ 被小周截留10年
                                         ↓
小周手中的信（收件人"小玲"）= 林小玲 = 林小姐
                                         ↓
阿蘅的祖父 → 提到陈伯（独臂恩人 / 省城医生）
                                         ↓
老吴 → 在火车站遇到白发阿蘅拍照（阿蘅二次来访）
                                         ↓
时先生 → 当铺创立者，120年守护七粒沙 = 七段遗憾
                                         ↓
时间旅人 → "第八粒沙" = 所有未被选择的选择的总和
```

### 3.3 记忆场景设计

每段记忆遵循「进入 → 对话 → 抉择 → 结果 → 离场」五段式结构：

```
phase: 'enter'  →  粒子特效 + 记忆标题淡入（60帧）
phase: 'dialog'  →  逐字打字 + 说话者标签
phase: 'choice'  →  选项列表（可能含隐藏选项，由数据驱动注入）
phase: 'result'  →  涟漪特效 + 结果对话
phase: 'exit'    →  条件离场对话检测 + 返回当铺
```

**记忆停留数据追踪：**
- `stats.memoryStayFrames[clientId]`：在每个角色记忆中的停留帧数
- `stats.choiceHesitateFrames`：选择犹豫帧数
- 用途：解锁"父亲背影""诗人"等时间成就

### 4.4 隐藏事件系统（12+ 事件）

**设计原则：** 隐藏事件由数据配置文件定义，触发逻辑通用化。增加新隐藏事件无需修改代码，只需在 `HIDDEN_EVENTS` 中添加条目。

```javascript
// HIDDEN_EVENTS 数据结构
const HIDDEN_EVENTS = {
  [eventId]: {
    id: string,
    triggerType: 'shop_enter' | 'idle_in_shop',  // 触发时机
    conditions: {               // 条件（AND 关系）
      ripples?: string[],       // 当前周目涟漪
      metaRipplesSeen?: string[], // 跨周目涟漪
      purchasedItems?: string[],
      completedClients?: string[],
      metaEndingSeen?: string[],
      metaEndingNotSeen?: string[],
      metaTrueEndingSeen?: boolean,
      metaMaxDays?: number,
      metaAllCollectibles?: boolean,
      metaHiddenEventSeen?: string[],
      metaAnyRippleSeen?: string[][],  // 每组至少一个
    },
    dialogs?: Dialog[],
    rewards?: { time?: number, collectible?: string },
    collectibleReward?: string,
    triggersEnding?: string
  }
}
```

## 五、结局设计（6+2 种）

### 5.1 结局类型

| 结局 | 触发条件 | 核心主题 |
|------|----------|----------|
| 晨光（ending_close） | 完成所有客人 + 选择关闭当铺 | 告别与新生 |
| 永恒（ending_stay） | 完成所有客人 + 选择继续经营 | 守护与传承 |
| 沙尽（time_out） | 时间碎片耗尽 | 遗憾与未完待续 |
| 闭环（ending_loop） | 五条涟漪交汇 + 神秘老人第三选项 | 因果之网 |
| 第七粒沙（ending_seventh_sand） | 5次时先生 + 全收藏品 + 双选择 | 真结局：时间的终章 |
| 第八粒沙（ending_eighth_sand） | 真结局 + 30天time_out + 全收藏品 | 隐藏终章：所有未被选择的选择 |

### 5.2 结局解锁依赖树

```
陈伯(2选)
林小姐(2选,蝴蝶阻断可选)
小周(2选)
阿蘅(2选)      ─┐
老吴(2选)       │
神秘老人(1选)  ─┤─ 神秘老人第三选项 → 闭环结局
                │
时先生(2选) ───┘── 时先生双选 → 真结局条件之一
                              ↓
                    五重门（全结局） → 第七粒沙
                              ↓
                    + 特殊条件 → 第八粒沙（隐藏终章）
```

## 八、数值经济

### 8.1 时间收入

### 8.3 经济模型分析

**健康状态（正常流程）：**
- 初始：100碎片
- 每日消耗：5
- 平均收入：~35/客人 × 7人 = 245
- 可持续天数：约 15–20 天（体验完整故事线）

**极限压缩（最低消耗）：**
- 30天时限可通过打磨沙漏（+50）延展
- 隐藏事件和共振可补充碎片缺口
- 守财奴成就（不购买任何升级通关）理论上可达

## 附录 B：核心数据结构速查

```
GameData.CLIENTS[clientId]
  ├── id, name, sprite, unlockDay, requiredShopLevel
  └── metaCondition: 'shixiansheng' (特殊解锁)

GameData.STORIES[clientId]
  ├── greeting / greeting_normal / greeting_butterfly
  ├── branchingGreeting: { baseGreeting?, introDialogs?, branchChoice[] }
  ├── memory: { title, description, dialogs[], choices[], hiddenChoice? }
  ├── conditionalDeparture[]: { afterRipple, metaRipplesSeen?, dialogs }
  └── ending_*: 各结局对话

GameData.BUTTERFLY_EFFECTS[effectId]
  ├── type: 'block_and_replace' | 'greeting_variant'
  ├── triggerRipple, targetClient
  └── butterflyDialogs / greetingKey

GameData.RESONANCE_EFFECTS[resId]
  ├── requiredRipples[], additionalCondition?
  ├── bonusTime, collectibleReward?

GameData.HIDDEN_EVENTS[eventId]
  ├── triggerType, conditions{}, dialogs?, rewards?
  └── triggersEnding?

GameData.ACHIEVEMENTS[achId]
  ├── name, description, category, rarity
  ├── hidden?, revealedDesc?
  └── reward?: { type: 'time'|'title'|'sprite'|'theme', value/amount }

MetaSave (cross-playthrough)
  ├── achievements, endingsSeen, allRipplesSeen, allChoicesMade
  ├── allResonancesSeen, collectibles, hiddenEventsTriggered
  └── shixianshengChoices, trueEndingSeen, maxDaysSurvivedTimeOut
```
