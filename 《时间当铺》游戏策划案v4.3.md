> 叙事解谜 / 像素风 / HTML5 Canvas
> 版本：v4.3（情感深化版 — 角色重逢结局 + 新角色"失忆者" + 时间压力机制 + 每日访客队列）

## 一、项目概述

### 1.1 一句话定位
经营一家收购"时间"的当铺，通过改写客人的过去来改变现在。

**差异化标签**：跨时空蝴蝶效应 × 数据驱动叙事 × 像素风叙事解谜

### 1.2 类型与风格
| 维度 | 内容 |
|------|------|
| 类型 | 叙事解谜 + 经营模拟 |
| 画面 | 像素风（16×20精灵，程序化背景） |
| 平台 | HTML5 Canvas，纯浏览器运行，无需安装 |
| 技术栈 | 原生 JavaScript，无框架依赖 |
| 存档 | localStorage（局内存档 + 跨周目 MetaSave 持久化）|

### 1.3 核心体验
玩家扮演一家神秘当铺的老板。客人以"记忆片段"作为抵押，换取改变命运的机会。进入客人的记忆，在关键节点做出选择——每个选择产生"时间涟漪"，跨越时空影响其他客人的命运。

### 1.4 核心设计亮点（v4.3 新增）

**亮点一：角色重逢与 After 结局**
陈波（chenbo）故事线在 v4.2 完成主结局后，v4.3 新增"重逢"（reunion）节点与"之后"（after）节点。玩家在完成该角色完整故事后，可体验重逢与日常生活后续，打通角色弧线的最后一环。

**亮点二：新角色——失忆者（shiyuren）**
v4.3 新增隐藏角色"失忆者"，通过收集特定涟漪组合（陈伯学医涟漪 + 林小姐完成 + 林小小完成）解锁。失忆者的故事线围绕记忆碎片与身份认同展开，是游戏主题（时间与记忆）的核心化身。

**亮点三：时间紧迫度机制**
每日结束扣减时间碎片从 5 增加到 8。同时新增"时间紧迫度"（timePressure）系统——当剩余天数 ≤ 5 时，背景音乐渐变为紧迫版本，UI 显示警示色。提升玩家的决策紧迫感。

**亮点四：每日访客队列**
每日最多接待 3 位客人。超过上限时新客人进入"明日队列"。队列中的老客户（下周回归的已接待角色）拥有优先权，使每日接待决策更具策略性。

## 二、核心机制（v4.3 变更）

### 2.1 时间交易循环（v4.3 更新）

```
每日开始
  ↓
读取访客队列（最多3人，老客户优先）
  ↓
接待客人 → 进入记忆 → 二选一抉择 → 产生涟漪 → 获得时间碎片
  ↓
当日接待已达3人 → 剩余客人进入明日队列
  ↓
结束今天 → 时间碎片 -8 → 时间紧迫度检测
  ↓
时间紧迫度 = 紧迫（≤5天）→ BGM切换 + UI警示
  ↓
30天时限内完成所有客人故事 → 触发结局
```

**每日访客队列规则：**
- 每日最大接待量：3 人
- 超出上限的客人进入"明日队列"
- 明日队列中，已有记录的返回客人（下周回归的老客户）优先于新客人
- 每个新周目重置队列

### 2.2 数据驱动的叙事系统（不变，详见 v4.2）

**新增数据节点类型（v4.3）：**

```javascript
// reunion 节点：角色重逢
clientData.reunion = {
  greeting: "...",
  dialogue: [...],
  choices: [...],
  afterScene: "after_scene_id"
}

// after 节点：角色日常生活后续
clientData.after = {
  greeting: "...",
  dialogue: [...],
  condition: { ripples: ["chen_doctor", "linxiaojie_complete", "linlin_complete"] }
}

// 新角色 shiyuren（失忆者）
const SHIYUREN_DATA = {
  id: "shiyuren",
  name: "失忆者",
  sprite: "shiyuren",
  greeting: { normal: "...", butterfly: "..." },
  storyline: "forget",
  unlockCondition: {
    ripples: ["chen_doctor", "linxiaojie_complete", "linlin_complete"],
    type: "AND"
  },
  firstVisit: { ... },
  reunion: { ... },
  after: { ... }
}
```

### 2.3 蝴蝶效应系统（不变，详见 v4.2）

**v4.3 新增涟漪用途：**
| 涟漪ID | 用途 | 影响 |
|--------|------|------|
| `chenbo_reunion` | 解锁陈波重逢 | 触发陈波与林小姐的重逢场景 |
| `shiyuren_unlock` | 解锁失忆者 | 失忆者出现在明日队列 |
| `shiyuren_memory_restore` | 记忆恢复 | 失忆者结局分支 |

### 2.4 跨周目 MetaSave 系统（v4.3 更新）

**v4.3 MetaSave 新增字段：**
```javascript
MetaSave {
  // v4.2 字段...
  chenboReunionSeen: false,      // 陈波重逢结局
  chenboAfterSeen: false,        // 陈波 After 结局
  shiyurenUnlocked: false,       // 失忆者解锁
  shiyurenEndingsSeen: {},       // 失忆者结局
  dailyQueueSeen: [],            // 今日已见队列
  shiyurenTrackCompleted: false // 失忆者故事线完成
}
```

### 2.5 时间紧迫度系统（v4.3 新增）

**紧迫度等级：**
| 剩余天数 | 紧迫度 | 效果 |
|----------|--------|------|
| > 10 天 | 正常 | 无特殊效果 |
| 6–10 天 | 注意 | UI 显示淡橙色边框 |
| ≤ 5 天 | 紧迫 | BGM 切换为紧迫版，UI 深红色警示，时间碎片图标闪烁 |

**BGM 切换逻辑：**
```javascript
// 当 timePressure 进入"紧迫"状态
if (remainingDays <= 5 && currentBGM !== 'urgent') {
  fadeOutCurrentBGM(1000);
  playBGM('urgent_shop');
}
```

### 3.2 陈波故事线（v4.3 扩展）

**v4.2 结局 → v4.3 重逢：**
玩家在 v4.2 达成陈波"重逢"结局（ripple: `chenbo_reunion`）后，新周目陈波再次来访时触发 `reunion` 节点。

```
v4.2 结局节点
    ↓
reunion：林小姐与陈波在小镇重逢
  - 陈波："我记得那家当铺……"
  - 林小姐出现客串："是他吗？"
  - 选择：让他们相认 / 让他们各自离开
    ↓
after 节点（后续日常生活）
  - 陈波与林小姐的日常生活片段
  - 展示时间改变后的平行人生
  - 触发收藏品："重逢之证"
```

### 3.3 失忆者故事线（v4.3 新增）

**解锁条件：**
同时满足以下三个条件，失忆者在明日队列中出现：
1. `chen_doctor` 涟漪存在（陈伯选择学医）
2. `linxiaojie_complete` 完成（林小姐故事线完成）
3. `linlin_complete` 完成（林小小故事线完成）

**故事线设计：**
失忆者是一个失去所有记忆的角色，他的存在本身是时间线紊乱的产物。

```
第一幕：陌生人来访
  - 失忆者出现在当铺，说"我不记得我是谁"
  - 玩家可选择帮他找回记忆 / 让他离开
    ↓
第二幕：记忆碎片
  - 失忆者多次来访，每次提供不同记忆碎片
  - 记忆碎片与其他角色的涟漪相关联
  - 收集足够碎片后，解锁"记忆恢复"结局
    ↓
第三幕：真相
  - 失忆者的真实身份揭示
  - 原来他是"时间"本身的化身
  - 结局分支：接受身份 / 放弃记忆作为普通人
    ↓
After 节点
  - 展示失忆者选择后的日常生活
```

## 七、测试要点（v4.3）

1. **每日队列验证**：连续接待 3 人后第 4 人进入队列，第 2 天队列优先出现老客户
2. **时间紧迫度验证**：剩余 5 天时 UI 变红；剩余 10 天时 UI 变橙
3. **时间碎片消耗验证**：每日结束确认扣除 8 而非 5
4. **失忆者解锁验证**：三个条件同时满足后新周目第 1 天出现
5. **陈波重逢验证**：在有 `chenbo_reunion` 涟漪的新周目中验证重逢节点触发
6. **BGM 切换验证**：进入紧迫状态后 BGM 平滑过渡到紧迫版
7. **涟漪传播验证**：失忆者记忆碎片与其他角色涟漪正确联动
8. **localStorage 验证**：MetaSave 正确记录所有 v4.3 新字段
